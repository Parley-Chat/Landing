Security Model

The server sees
ciphertext only.

  • RSA-OAEP key exchange ‚Äî private key never transmitted
  • AES-GCM-128 message encryption ‚Äî server stores only ciphertext
  • RSA-PSS message signing ‚Äî tampering is cryptographically detectable
  • Hourly key rotation ‚Äî forward secrecy limits breach impact
  • Encrypted key file backup for account recovery
  • Session management ‚Äî revoke device access at any time
  • MIT licensed ‚Äî read and audit every line of code

Parley Chat uses hybrid encryption: RSA-OAEP exchanges an ephemeral AES-GCM shared key per channel. Messages are encrypted with that key and signed with RSA-PSS before leaving your device. The server receives and stores only ciphertext.

Shared keys expire every hour. Even if a key is compromised, only one hour of messages is exposed ‚Äî forward secrecy limits the impact of any breach to a short window.
